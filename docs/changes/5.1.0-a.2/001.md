# 变更 001: 修复顶层函数类型注册

## 问题描述
顶层函数注册到符号表时使用了返回类型而不是函数类型，导致函数指针类型检查失败。

## 影响范围
- `src/semantic/analyzer.rs`: 顶层函数注册逻辑

## 技术细节
在注册顶层函数时，原本代码只存储了返回类型：
```rust
self.current_scope.borrow_mut().define(
    &func.name,
    func.return_type.clone(),
    ...
);
```

修复后存储完整的函数类型：
```rust
let func_type = Type::Function(Box::new(FunctionType {
    params: func.params.iter().map(|p| p.param_type.clone()).collect(),
    return_type: Box::new(func.return_type.clone()),
    is_static: true,
}));
self.current_scope.borrow_mut().define(
    &func.name,
    func_type,
    ...
);
```

## 复杂度分析
- 时间复杂度: O(n)，n为参数数量
- 空间复杂度: O(1)

## 验证
- 测试用例: `test_callback_pattern` 中传递顶层函数作为回调参数
- 所有 8 个 FFI 测试通过
