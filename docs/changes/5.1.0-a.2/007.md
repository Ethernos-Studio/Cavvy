# 变更 007: 修复 c_string 类型定义冲突

## 问题描述
`c_string` 同时作为内置类型和类型别名定义，导致解析冲突。

## 影响范围
- `caylibs/std/ffi.cay`: 移除冲突的别名定义
- `src/lexer/mod.rs`: 添加 `c_string` token
- `src/parser/types.rs`: 添加 `c_string` 类型解析

## 技术细节

### 1. 标准库修改
移除 `std/ffi.cay` 中的冲突定义：
```rust
// 移除了以下行：
// alias c_string = c_char*;           // C 字符串 (char*)
// c_string 现在是内置类型，不需要别名定义
```

### 2. 词法分析器修改
添加 `c_string` token：
```rust
"c_string" => Token::CString,
```

### 3. 解析器修改
添加 `c_string` 类型解析：
```rust
Token::CString => {
    input.next();
    Ok(Type::Pointer(Box::new(Type::CChar)))
}
```

## 复杂度分析
- 时间复杂度: O(1)
- 空间复杂度: O(1)
