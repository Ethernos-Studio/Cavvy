# 变更 003: 修复顶层函数作为函数指针传递

## 问题描述
顶层函数作为函数指针参数传递时，代码生成器未正确处理，导致 LLVM IR 生成错误。

## 影响范围
- `src/codegen/expressions/identifier.rs`: `generate_identifier` 方法
- `src/codegen/context.rs`: 新增 `get_top_level_function_type` 方法

## 技术细节

### 1. 标识符生成修复
在 `generate_identifier` 中添加顶层函数检测：
```rust
// 检查是否是顶层函数 - 返回函数指针
if self.is_top_level_function(name) {
    let func_ptr_type = self.get_top_level_function_type(name);
    let llvm_func_type = self.type_to_llvm(&func_ptr_type);
    let func_name = format!("__toplevel_{}", name);
    return Ok(format!("{} @{}", llvm_func_type, func_name));
}
```

### 2. 新增类型获取方法
```rust
pub fn get_top_level_function_type(&self, func_name: &str) -> crate::types::Type {
    if let Some(func) = self.top_level_functions.iter().find(|f| f.name == func_name) {
        crate::types::Type::Function(Box::new(crate::types::FunctionType {
            params: func.params.iter().map(|p| p.param_type.clone()).collect(),
            return_type: Box::new(func.return_type.clone()),
            is_static: true,
        }))
    } else {
        // 默认返回 int () 类型
        crate::types::Type::Function(Box::new(crate::types::FunctionType {
            params: vec![],
            return_type: Box::new(crate::types::Type::Int32),
            is_static: true,
        }))
    }
}
```

## 复杂度分析
- 时间复杂度: O(n)，n为顶层函数数量
- 空间复杂度: O(1)

- 测试用例: `test_callback_pattern` 中传递 `compare_ints` 给 `bubble_sort`
