# 变更 002: 修复顶层函数参数类型保存

## 问题描述
顶层函数参数在代码生成阶段没有保存 Cavvy 类型信息，导致函数指针变量识别失败。

## 影响范围
- `src/codegen/generator.rs`: `generate_top_level_function` 方法

## 技术细节
在生成顶层函数时，参数类型只保存到 `var_types`（LLVM 类型），未保存到 `var_cay_types`（Cavvy 类型）：

```rust
// 修复前
self.var_types.insert(param.name.clone(), param_type);

// 修复后
self.var_types.insert(param.name.clone(), param_type);
// 同时保存Cavvy类型用于函数指针识别
self.var_cay_types.insert(param.name.clone(), param.param_type.clone());
```

这使得 `get_variable_type()` 能够正确识别函数指针参数类型。

## 复杂度分析
- 时间复杂度: O(1)
- 空间复杂度: O(1)

## 验证
- 测试用例: `test_callback_pattern` 中 `bubble_sort` 函数调用 `cmp` 参数
